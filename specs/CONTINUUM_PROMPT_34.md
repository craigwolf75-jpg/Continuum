## 3. Two naming collisions to settle before the room

SIGMA-HR versus SIGMA-RH. This spec says SIGMA-HR throughout; Craig's integration blueprint, adopted as the integration bible, says SIGMA-RH throughout, and the product GardaWorld runs is SIGMA-RH (the RH is the French Ressources Humaines). Saying the client's own system's name wrong in front of their executives is the kind of small wound that costs a room, so the artifact ships with the system name as a single constant at the top of the script, defaulted to the spec's SIGMA-HR and flipped everywhere by one edit. Confirm which name GardaWorld's own people use and set it before Wednesday.

CRTW's expansion. This spec defines CRTW as the Connected Return-to-Work platform; Craig's PRD defines CRTW as Corporate Recovery, Training and Workforce Intelligence, the analytics layer. Two expansions of one acronym inside one venture will eventually collide in a document a client reads. The page follows this spec's expansion, which is frankly the better one for a sales room, but the venture should rule on one expansion platform-wide, and the companion flags it for the next Craig exchange.

## 4. Presenter guidance

The overall message rides in the notes on every screen and closes the loop the way the spec asks: GardaWorld does not need to replace its system of record; Continuum connects around the existing process and manages the active recovery, functional progress, modified work, and the return journey. The strongest moment to slow down is screen four, which shows the employer view by naming what is deliberately absent from it, and the second strongest is screen seven, where the honesty about the write path (prepared and ready, method confirmed in technical discovery) is itself the credibility argument in front of an information-technology audience.
