Design reference

Continuum surface inventory

The build reference for the connected demo, drawn from the client MVP wireframe reference (v2). Each surface reads live from one shared store. This gallery is gated and not indexed. It documents the build; it is not the product entry.

Worker App

Consent gate, injury summary, twice daily slider check-in, history, and a light duty checklist. One phone column.

/app

Employer dashboard

Functional status only. KPI row and worker table by recovery status. Never pain or mobility scores, diagnosis, photos, or free text.

/hub · Employer

Light Duties (HSE)

Assign modified duties within published restrictions. Scores visible to HSE for operational need. Restrictions read only to HSE.

/hub · HSE

Nexus Health

Clinical control center. Full medical detail, monitoring, escalation alerts, clearance actions, and the fitness for work form.

/hub · Nexus

WCB view

Read only claims and the three milestone notifications. Document lifecycle tracking only. No live WCB connection.

/hub · WCB

GardaWorld demo runner

The fourteen-scene presenter instrument: narration, working vignettes, presenter notes with the protective sentences, who-becomes-smarter chips, and live-system pointers to the real portals. Print produces the full script handout.

/garda-demo

Claims system panel

Illustrative mock of Continuum Recovery Intelligence embedded in a claims system. Functional signals only, live from the demo bridge. Pilot integration is intake level, not bidirectional.

/sigma-panel
